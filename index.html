Areas de git

workin Directory

Es el area de trabaja, donde rara vez vamos a utilizar la linea de comando, aquella donde realizamos las modificaciones de nuestro proyecto tales como la creacion, ediciona y elminacion de archivos. El unico comando necesario sera enfasis en git init para iniciar nuestro proyecto gestionado con la herramienta git

Staging Area (INDEX)

Es el area de control, donde estaremos agregando los archivos a los que queremos hacerles seguimientos, una vez hecho esto podemos realizar la confirmacion de cambios para poder recuperar dichas modificaciones en caso de ser necesario. Cuando agregamos un archivo al INDEX git realiza una captura del codigo(snaphot) que permite identificar las lineas agregadas o eliminadas.

Repositorio (Local/REmote)

Area donde se almacenan las confirmaciones de las captura en formato blob para que puedan ser utilizadas al momento de movenrnos en nuestro proyecto y restablecer el mismo. Git gestiona los cambios localmente, pe podemos enviar el historial de cambios a una direccion externa, tambiien conocida como respositorio remoto.
